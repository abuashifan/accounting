Wiring auto-post ke flow create journal (backend business logic)

File utama rules: backendAPI/app/Domains/Accounting/Services/JournalService.php
Pada create():
Buat jurnal seperti biasa (draft), log audit journal.created.
Jika setting journals.auto_post true:
otomatis menjalankan logic posting internal postEntry() sehingga status menjadi posted dan log audit journal.posted.
Catatan behavior penting:
Dengan implementasi sekarang, user yang hanya punya permission journal.create (tanpa journal.update) tetap bisa membuat jurnal yang langsung posted saat auto-post aktif, karena posting dilakukan di dalam service create. Ini sesuai requirement “autopost global” tapi perlu disadari dampak kontrol internalnya.
Posting tetap menjalankan checks:
status harus draft
period tidak boleh closed
debit = credit (balanced)
update status ke posted dan audit log.

PERUBAHAN UX/API YANG MENJADI “KONTRAK” UNTUK FRONTEND TERPISAH NANTI

Frontend tidak boleh mengirim account code ke lines[].account_id. Harus mengirim accounts.id.
Untuk menampilkan pilihan akun, frontend gunakan:
GET /api/accounts (auth required)
Untuk setting auto-post:
GET /api/settings/journals
PUT /api/settings/journals dengan payload boolean
Behavior create journal:
Default -> status draft
Jika setting auto-post aktif -> status posted segera dan masuk laporan
Posting manual tetap tersedia:
POST /api/journals/{id}/post
Validasi inti tetap di backend requests:
StoreJournalRequest: lines.\*.account_id exists accounts,id; debit/credit numeric min 0; lines min 2; period exists & period lock rules.
CATATAN OPERASIONAL / TROUBLESHOOTING (penting agar tidak miss komunikasi)

jalankan: php artisan db:seed --class=Database\\Seeders\\RbacSeeder
sudah ditambahkan fallback: role admin boleh manage settings walau permission settings.manage belum tersync.
