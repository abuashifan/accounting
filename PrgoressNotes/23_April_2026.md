## DoD / Definition of Done — Inventory Purchase + Sales via Invoice (Laravel `backendAPI`)

### 1) Data & Model

- Tabel `invoice_lines` ada dan terhubung ke `invoices`, `items`, `warehouses`, menyimpan `quantity`, `unit_price`, `line_total`.
  - `backendAPI/database/migrations/2026_04_23_000200_create_invoice_lines_table.php`
- Kolom `invoices.posted_at` ada untuk menandai invoice sudah diposting.
  - `backendAPI/database/migrations/2026_04_23_000100_add_posted_at_to_invoices_table.php`
- Model + relasi:
  - `InvoiceLine` model tersedia.
  - `Invoice->invoiceLines()` tersedia.

### 2) Aturan Auto Post (mengikuti aturan yang sudah berlaku)

- `JournalService::create(...)` mendukung override auto-post dan default mengikuti setting `journals.auto_post`.
- Ada jalur internal `postAsSystem()` untuk auto-post tanpa butuh permission `journal.update` (konsisten dengan auto-post saat create).

### 3) Pembelian (menambah stok)

- Endpoint pembelian stok tersedia: `POST /api/stocks/purchase`.
- Purchase:
  - update `stock_balances`
  - insert `stock_movements` tipe `purchase`
  - create journal pembelian (posted/draft mengikuti `journals.auto_post`)

### 4) Penjualan pakai Invoice (mengurangi stok saat invoice posted)

- Endpoint sales invoice:
  - `POST /api/invoices` (create sales invoice + invoice lines)
  - `POST /api/invoices/{id}/post` (posting: kurangi stok + jurnal HPP)
  - `GET /api/invoices`, `GET /api/invoices/{id}`
- Saat posted:
  - jurnal: `Piutang -> Penjualan` dan `HPP -> Persediaan`
  - qty stok berkurang
  - `stock_movements` tipe `sale` tercatat dengan `reference_type=invoice` & `reference_id=invoice_id`
- Perilaku mengikuti `journals.auto_post`:
  - `journals.auto_post=true`: invoice langsung posted + stok langsung berkurang + `posted_at` terisi
  - `journals.auto_post=false`: invoice & jurnal draft, stok belum berubah sampai posting manual

### 5) History Produk (keluar/masuk barang, selain Stock Card)

- Endpoint: `GET /api/items/{id}/history`
- Output minimal: `tanggal`, `nomor transaksi`, `sumber transaksi`, `qty_in/qty_out` (+ warehouse)

### 6) Debug UI (untuk verifikasi cepat)

- Debug Invoice bisa buat sales invoice dengan `Item/Warehouse/Qty/Unit Price`.
- Tombol `Post` muncul jika jurnal masih draft (auto_post off).

### 7) Testing

- `php artisan test`: 31 passed.

  - Auto journal invoice/payment mengikuti setting + permission (`AutoJournalEngineTest`).
  - Sales invoice inventory: draft saat auto_post off, posting mengurangi stok & menambah jurnal HPP, auto-post saat auto_post on (`SalesInvoiceInventoryTest`).

## DoD Tambahan — Modul Pembelian (Faktur Pembelian + Pembayaran Pembelian)

### Data & Model

- Ada tabel:
  - `purchase_invoices` (punya `posted_at`, `journal_entry_id`, `amount`, `paid_amount`, `status`)
  - `purchase_invoice_lines` (item, warehouse, qty, unit_cost, line_total)
  - `purchase_payments` (payment_no, purchase_invoice_id, journal_entry_id)

### API & Business Rules

- Faktur pembelian:
  - `POST /api/purchase-invoices` membuat jurnal draft `Persediaan(D) - Utang(C)` dan menyimpan lines.
  - `POST /api/purchase-invoices/{id}/post` melakukan posting: stok bertambah + movement `purchase` + jurnal menjadi posted.
  - Jika `journals.auto_post=true`, maka saat create invoice akan langsung melakukan posting (tanpa butuh permission `journal.update`).
- Pembayaran pembelian:
  - `POST /api/purchase-invoices/{id}/payments` membuat jurnal `Utang(D) - Kas/Bank(C)` dan update `paid_amount/status`.
  - Perilaku posted/draft mengikuti `journals.auto_post`.

### Debug UI

- Tambah menu dan halaman Debug:
  - `Purchase Invoice` list + create + tombol `Post` (jika draft dan auto_post off).
  - Halaman `Pay` untuk record pembayaran pembelian.

### Testing

- `PurchaseInvoiceInventoryTest` cover:
  - auto_post off: draft -> posting menambah stok + jurnal posted.
  - auto_post on: create langsung posted + stok bertambah.

  - `destroy(int $id, InvoiceService $service)`
- Implement logic di `backendAPI/app/Domains/Accounting/Services/InvoiceService.php`:

    - Update harus update jurnal draft `AR vs Revenue` (dan invoice lines/amount).
    - Delete: void journal draft (atau delete journal entry kalau pattern-nya mengizinkan), lalu delete invoice + lines.
